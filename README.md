# US Migration Explorer

Interactive explorer for IRS Statistics of Income state-to-state migration data,
broken down by AGI bracket and age of the primary filer.

The popular narrative about state-to-state migration ("people leaving CA/NY for
FL/TX because of taxes") collapses several very different stories — retirees
moving for weather, working-age households moving for affordability, high
earners moving for tax rates. The IRS data lets you separate these. This tool
makes that separation legible.

## Try it

No build step. Clone, serve, open:

```bash
git clone https://github.com/ukogan/us-migration-explorer.git
cd us-migration-explorer/mockup
python3 -m http.server 5180
# open http://localhost:5180/SOI%20Migration%20Explorer.html
```

The data is checked in (`mockup/data-real.js`, 7.7 MB) so a clone is all you need.

## Three views, one selected state

1. **Origin × Destination matrix** — 51×51 heatmap of state-pair flows. Totals
   only; brackets are not available at the pair level.
2. **Top flows Sankey** — the same pair data as ribbon flows, sorted by volume.
3. **AGI × Age cross-tab** — for the selected state, a heatmap of every
   AGI bracket × age bracket cell. Fully joint — "65+ AND $200K+" is supported.

A shared year slider (2011–2022) and direction toggle (inflow / outflow / net)
drive all three views. State selection is shared too — click a state in any
view to update the others.

## The data constraint that shapes the UI

The IRS publishes two files that do not join:

- **State-pair flows** (`stateinflow / stateoutflow CSVs`) — every origin ×
  destination pair, but no bracket columns. Totals only.
- **Gross Migration file** (`YYinmigall.csv`) — per-state aggregates only (no
  destination breakdown), but fully cross-tabbed AGI × age × five flow categories.

So you can ask "where did Californians move?" (matrix / Sankey) or "what was
the AGI × age mix of leavers from California?" (cross-tab), but not "how many
65+ $200K+ filers moved from CA to FL specifically" — that joint distribution
isn't published. The UI surfaces this constraint instead of pretending it's
not there.

See [`docs/data-sources.md`](docs/data-sources.md) for the full schema and
provenance.

## Refreshing the data

When the IRS publishes a new filing year, add the year code to `YEAR_PAIRS`
in `scripts/build_data.py`, drop the new CSVs into `data/raw/<year>/`, and
re-run `python3 scripts/build_data.py`. Full instructions in
[`docs/rebuilding-data.md`](docs/rebuilding-data.md).
